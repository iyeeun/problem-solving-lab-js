## Deep Dive

### 1. 핵심 질문

> 이 장은 결국 무엇을 설명하려는가?

- 함수 실행이 끝난 이후에도 외부 변수에 접근할 수 있는 이유와 그 구조(클로저)를 설명한다
- 이러한 구조가 실제로 어떤 상황에서 활용되는지를 설명한다

### 2. 한 줄 요약

> 이 장의 핵심을 한 문장으로 말하면?

- 클로저란 내부 함수가 외부 스코프의 변수를 참조하고, 해당 함수가 외부로 전달될 때 함수 실행 이후에도 그 변수에 계속 접근할 수 있는 현상이다.

### 3. 개념 구조 정리

#### 📚 핵심 개념

- 클로저
- Lexical Environment
- 스코프 체인

#### 🔗 개념 연결 구조

- 클로저 : 내부 함수가 외부 변수에 접근 가능한 상태
- 발생 조건
  - 내부 함수가 외부 변수 참조
  - 해당 함수가 외부로 전달됨 (return / 콜백)
- 동작 방식
  - 함수가 선언될 때 Lexical Environment 생성
  - 내부 함수는 해당 환경을 참조
  - 함수 실행 종료 후에도 참조가 남아 GC 대상에서 제외됨
    -> 외부 함수가 끝났어도 변수 유지됨

### 4. 나의 언어로 설명해보기

클로저는 내부 함수가 외부 스코프의 변수에 접근 가능한 상태를 말한다.  
일반적으로 함수 실행이 종료되면 내부 변수도 사라지지만,  
내부 함수가 외부 변수를 계속 참조하고 있기 때문에 GC 대상이 되지 않는다.

### 5. 헷갈렸던 부분 / 오해했던 부분

#### ❓ 처음에 이해 안 됐던 점

- 클로저와 스코프 체인의 차이
- 모든 내부 함수가 클로저라고 생각했다.

#### 💡 지금은 이렇게 이해했다

- 스코프 체인은 단순히 변수를 찾는 규칙이고, 클로저는 유지되는 현상이다.
- 내부 함수가 외부로 전달되고, 외부 변수를 참조할 때만 클로저가 발생한다.

### 6. 코드로 확인하기

#### 클로저 기본 예시

```js
function outer() {
  let a = 1;

  return function () {
    return ++a;
  };
}

const fn = outer();

console.log(fn()); // 2
console.log(fn()); // 3
```

- 결과 : 2, 3
- 이유 :
  outer는 이미 실행 종료됨
  하지만 내부 함수가 a를 계속 참조 → 클로저 발생

#### 클로저 미발생 예시

```js
function outer() {
  let a = 1;

  function inner() {
    console.log(a);
  }

  inner();
}

outer();
```

- 결과 : 1
- 이유 :
  단순 스코프 체인 동작
  외부로 함수가 전달되지 않으므로 클로저가 유지되지 않음

### 7. 실무 연결 포인트 💼

#### 🔎 이 개념이 실제로 왜 중요한가?

- 상태를 안전하게 유지할 수 있음 (전역 변수 없이)
- 이벤트 핸들러, 비동기 처리에서 데이터 유지
- 디바운스, 커링, partial 같은 패턴의 핵심

#### 🛠 앞으로 코드 작성 시 적용할 점

- 전역 변수 대신 클로저로 상태 관리하기
- 이벤트 핸들러에서 필요한 데이터 캡슐화하기
- 불필요한 클로저는 제거해서 메모리 관리하기
- 필요 없어진 참조는 null로 끊기

### 8. 면접 대비용 정리

#### ❓ 예상 질문

1. 클로저란 무엇인가요?

- 외부 함수에서 선언한 변수를 참조하는 내부 함수가 외부로 전달될 때,  
  외부 함수의 실행 컨텍스트가 종료된 이후에도 그 변수에 계속 접근할 수 있는 현상입니다.  
  JS 엔진이 내부 함수의 Lexical Environment 참조를 GC하지 않고 유지하기 때문에 발생합니다.

2. 클로저는 어떤 상황에서 활용되나요?

- 콜백 함수에서 외부 데이터를 안전하게 참조할 때, 정보 은닉(private 변수 구현), 부분 적용 함수, 커링, 디바운스/스로틀링 구현 등에 활용됩니다.

3. 클로저 사용 시 주의할 점은 무엇인가요?

- 클로저가 참조하는 변수는 GC되지 않으므로 메모리 누수가 발생할 수 있습니다.  
  더 이상 필요하지 않은 클로저는 참조 변수를 `null`로 재할당하여 명시적으로 참조를 끊어야 합니다.

4. 스코프 체인과 클로저의 차이는 무엇인가요?

- 스코프 체인은 함수 실행 중에 상위 스코프의 변수를 탐색하는 것입니다.  
  클로저는 외부 함수가 이미 종료된 이후에도 내부 함수가 그 환경을 기억하고 접근할 수 있는 현상을 말합니다.

## 최종 인사이트

클로저는 JS가 동작하는 방식에 의해 자연스럽게 생겨나는 현상이다.  
함수가 외부 스코프를 참조한 채로 전달될 수 있기 때문에, 이미 실행이 끝난 함수의 변수라도 계속 접근 가능한 상태가 유지된다.  
이러한 특성을 의도적으로 활용하면 상태 유지, 캡슐화, 함수형 패턴 구현이 가능하다.  
다만 클로저는 메모리를 의도적으로 붙잡는 것이기 때문에 생명주기를 명확히 관리하는 습관을 들여야 한다.
