# [모의고사](https://school.programmers.co.kr/learn/courses/30/lessons/42840)

## 문제 요약

- 3명의 수포자가 각자 반복되는 패턴으로 답을 찍음
- 정답 배열이 주어질 때 가장 많이 맞춘 사람을 반환
- 동점이면 오름차순

## 입력 구조

- answers.length: 최대 10000
- 값 범위: 1 ~ 5

## 문제 유형 분류

- 완전탐색
- 시뮬레이션

## 사고 과정

### 1. 왜 이 유형인가

- 주어진 값과 하나씩 비교해야 함

### 2. 왜 다른 접근은 안되는가

- n이 10000 미만으로 크지 않고, 패턴이 있기 때문에 시뮬레이션을 돌리기 쉬움

## 문제 풀이

- [42840.js (8분 소요)](./42840.js)

```
1. 각 수포자의 찍기 패턴을 정의
 -> 기본 단위만 두고 mod 연산을 통해 반복
2. 정답을 돌면서 맞는지 확인하여 점수 기록
3. 최종 점수를 기준으로 가장 많이 맞춘 사람 배열에 추가
```

## 해결 전략

### 알고리즘 선택 이유

- 패턴이 있기 때문에 해당 패턴을 정의 후 조건을 이용해서 시뮬레이션
- 반복 패턴 -> mod 연산

### 자료구조 선택 이유

- 단순 기록용이므로 배열 이용

## 시간/공간 복잡도

- 시간: O(n)
- 공간: O(1)

## JS 구현 포인트

- 2차원 배열로 정의하여 비교 로직도 단순화
