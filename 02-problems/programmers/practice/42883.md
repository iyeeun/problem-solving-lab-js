# [큰 수 만들기](https://school.programmers.co.kr/learn/courses/30/lessons/42883)

## 문제 요약

- 숫자 문자열에서 k개의 숫자를 제거 (순서는 유지)
- 남은 숫자로 만들 수 있는 가장 큰 수를 구하는 문제

## 입력 구조

- number: 길이 2 이상 1000000 이하의 문자열
- k: 1 이상 number 길이 미만

## 문제 유형 분류

- Greedy
- Stack

## 사고 과정

### 1. 왜 이 유형인가

- 앞자리가 클수록 전체 숫자가 커짐 -> 작은 숫자는 앞에 두지 않음
- 현재 숫자보다 작은 숫자는 제거 => Stack + Greedy

### 2. 왜 다른 접근은 안되는가

- 완전탐색: 조합 수가 매우 큼 (시간 초과)
- DP: 상태 수가 너무 많음
- 정렬: 순서를 유지해야 하므로 불가능

## 문제 풀이

- [42883.js (23분 소요)](./42883.js)

```
- 숫자를 하나씩 보면서 stack에 넣음
- 현재 숫자가 더 크면 이전 작은 숫자 제거
- k만큼 제거할 때까지 반복
- 남은 k는 뒤에서 제거
```

## 해결 전략

### 알고리즘 선택 이유

- 앞자리 숫자가 전체 값에 큰 영향
- 현재보다 작은 이전 값은 제거하는 것이 항상 이득

### 자료구조 선택 이유

- Stack(배열): 최근 값과 비교 및 제거가 효율적

## 시간/공간 복잡도

- 시간: O(N)
- 공간: O(N)

## JS 구현 포인트

- while 조건에서 k > 0 체크 필수
- 문자열을 직접 비교해도 문제 없음
- 마지막 k 처리 중요
