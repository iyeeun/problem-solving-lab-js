# [조이스틱](https://school.programmers.co.kr/learn/courses/30/lessons/42860)

## 문제 요약

- 초기 문자열은 전부 A
- 목표 문자열 name으로 만들기
- 가능한 조작:
  - 위/아래: 문자 변경
  - 좌/우: 커서 이동

- 최소 조작 횟수 구하기

## 입력 구조

- name: 길이 1 이상 20 이하
- 대문자 알파벳 문자열

## 문제 유형 분류

- Greedy
- 문자열 처리
- 완전탐색(부분)

## 사고 과정

### 1. 왜 이 유형인가

- 이 문제는 문자 변경과 커서 변경을 따로 봐야 함
- 문자 변경은 독립적 -> Greedy
- 커서 이동은 경로 선택하는 문제

### 2. 왜 다른 접근은 안되는가

- DFS: 모든 이동 경로 탐색 -> 불필요하게 큼
- DP: 상태 정의가 비효율적
- BFS: 상태 공간이 커짐

## 문제 풀이

- [42860.js (60분+ 소요)](./42860.js)

```
1. 각 문자에 대해 A -> 목표 문자 최소 이동 횟수 계산 (위/아래)
2. 커서 이동 최소값 계산
   - 기본: 오른쪽으로 끝까지 (n - 1)
   - 모든 위치 i에 대해:
     - i까지 이동
     - 이후 연속된 A 구간 탐색 (next)
     - A 구간을 스킵하는 두 가지 경우 계산
       - move1: 오른쪽 갔다가 돌아오기
       - move2: 왼쪽 먼저 갔다가 돌아오기
3. 최소 커서 이동 + 문자 이동 합산
```

## 해결 전략

### 알고리즘 선택 이유

- 문자 변경은 독립적인 Greedy
- 커서 이동은 전체 경로 최적화 필요

### 자료구조 선택 이유

- 별도 자료구조 없이 문자열 순회

## 시간/공간 복잡도

- 시간: O(N^2)
- 공간: O(1)

## JS 구현 포인트

- charCodeAt으로 문자 계산
- Z에서 A로 내려오는 케이스 고려
- while로 연속된 A 스킵
