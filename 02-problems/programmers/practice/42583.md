# [다리를 지나는 트럭](https://school.programmers.co.kr/learn/courses/30/lessons/42583)

## 문제 요약

- 트럭이 순서대로 다리를 건넌다
- 다리는
  - 최대 `bridge_length` 길이
  - 최대 `weight` 무게 제한
- 모든 트럭이 다 건너는 최소 시간

## 입력 구조

- `bridge_length`: 1 이상 10000 이하
- `weight`: 1 이상 10000 이하
- `truck_weights.length`: 1 이상 10000 이하

## 문제 유형 분류

- 큐
- 시뮬레이션

## 사고 과정

### 1. 왜 이 유형인가

- 시간에 따라 변하는 트럭을 큐를 이용해서 시뮬레이션 함

### 2. 왜 다른 접근은 안되는가

- 선택 문제 아님 (Greedy X)
- 상태 누적 구조 아님 (DP X)

## 해결 전략

### 알고리즘 선택 이유

- 시간 단위로 상태가 바뀜
- 조건에 따라 진입/이탈 결정

### 자료구조 선택 이유

- 배열로 다리를 모델링
- 큐 + 객체로 트럭이 나가는 시간을 관리

## 시간/공간 복잡도

- 시간: O(N \* bridge_length) / O(N)
- 공간: O(bridge_length) / O(N)

## JS 구현 포인트

- `shift()`로 큐 이동
- 시간 흐름은 `while`로 제어
