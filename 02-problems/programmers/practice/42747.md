# [H-Index](https://school.programmers.co.kr/learn/courses/30/lessons/42747)

## 문제 요약

- 연구자의 논문 인용 횟수가 담긴 배열이 주어질 때, H-Index를 구하는 문제
- H-Index: h번 이상 인용된 논문이 h편 이상이고, 나머지는 h번 이하 인용

## 입력 구조

- citations: 논문의 인용 횟수를 담은 배열
- 길이: 1 이상 1000 이하
- 각 원소: 0 이상 10000 이하

## 문제 유형 분류

- 정렬
- Greedy

## 사고 과정

### 1. 왜 이 유형인가

- 최대한 큰 h를 찾는 문제이므로 최적 선택을 순차적으로 확인
- 큰 값부터 확인하면 조건을 빠르게 만족하는 지점 발견 가능
- 따라서 내림차순 정렬 후 greedy하게 검사

### 2. 왜 다른 접근은 안되는가

- DP: 이전 상태를 기반으로 최적을 누적하는 구조가 아님
- BFS/그래프: 관계 탐색 문제가 아님
- Two Pointer: 정렬 후 특정 구간을 유지하는 구조가 아니라 단일 조건 비교 문제

## 문제 풀이

- [42747.js](./42747.js)

```
1. citations를 내림차순 정렬
2. i번째 논문까지 봤을 때 (i+1편)
3. citations[i] >= i+1이면 조건 만족
4. citations[i] < i+1 되는 순간, i가 H-Index
5. 끝까지 통과하면 전체 길이가 H-Index
```

## 해결 전략

### 알고리즘 선택 이유

- 가장 많이 인용된 논문부터 확인하면 현재까지 몇 편이 조건을 만족하는지 바로 판단 가능
- 조건이 깨지는 최초 지점이 H-Index

### 자료구조 선택 이유

- 배열 정렬만으로 충분

## 시간/공간 복잡도

- 시간: O(N log N)
- 공간: O(1)

## JS 구현 포인트

- i는 0부터 시작하므로 논문 개수는 `i + 1`
