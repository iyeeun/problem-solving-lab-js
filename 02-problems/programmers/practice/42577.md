# [전화번호 목록](https://school.programmers.co.kr/learn/courses/30/lessons/42577)

## 문제 요약

- 어떤 번호가 다른 번호의 접두어(prefix)이면 `false`를, 아니면 `true`를 반환

## 입력 구조

- `phone_book.length`: 1 ~ 1,000,000
- 각 문자열 길이: 1 ~ 20
- 중복 없음

## 문제 유형 분류

- 정렬
- 해시
- 문자열

## 사고 과정

### 1. 왜 이 유형인가

- 각 쌍에 대한 비교를 수행하는 것이 가장 쉬운 방법이지만 O(n^2)임
- 시간을 줄이기 위해서 필터링이 필요함
  - 정렬을 통해 접두어 관계인 문자열을 인접하게 위치시켜야 함

### 2. 왜 다른 접근은 안되는가

- n이 크기 때문에 완전 탐색은 불가능

## 해결 전략

### 알고리즘 선택 이유

- Set으로 prefix 존재 여부 빠르게 탐색
- **정렬하면 접두어 관계는 반드시 인접한 원소에서만 발생함**

### 자료구조 선택 이유

- `Set`: O(1) 탐색

## 시간/공간 복잡도

- 시간: O(N log N + N \* L^2) / O(N log N + N \* L)
- 공간: O(N) / O(1)

> L = 최대 문자열 길이 (20)

## JS 구현 포인트

- `slice()`로 prefix 생성
- 조기 종료 (early return)
- `Set`으로 빠른 존재 확인
- `startsWith()` 활용 (의도가 명확)
