# [타겟 넘버](https://school.programmers.co.kr/learn/courses/30/lessons/43165)

## 문제 요약

- 주어진 숫자 배열의 각 원소에 대해 더하거나 빼는 연산을 선택하여
- target 값을 만드는 경우의 수를 구하는 문제

## 입력 구조

- numbers: 길이 2 이상 20 이하의 정수 배열
- 각 원소: 1 이상 50 이하
- target: 1 이상 1000 이하

## 문제 유형 분류

- DFS / 완전탐색
- (확장) 백트래킹

## 사고 과정

### 1. 왜 이 유형인가

- 각 숫자마다 선택지는 2개 (+, -)
- 전체 경우의 수는 2^numbers.length

### 2. 왜 다른 접근은 안되는가

- Greedy: 현재 선택이 이후 결과에 영향을 주므로 적용 불가
- Two Pointer: 정렬 및 구간 문제 아님
- BFS: 가능하지만 DFS보다 상태 표현이 불편함

## 문제 풀이

- [43165.js](./43165.js)

```
- 각 단계마다 현재 숫자를 더할지 뺄지 선택
- 모든 경우를 탐색한 후 합이 target과 같으면 카운트
```

## 해결 전략

### 알고리즘 선택 이유

- 각 원소마다 두 가지 선택이 존재 -> 완전탐색 구조
- 모든 경우를 확인해야 정확한 답을 구할 수 있음

### 자료구조 선택 이유

- 별도의 자료구조 없이 sum 변수로 상태 압축
- 재귀 호출로 상태를 자연스럽게 표현

## 시간/공간 복잡도

- 시간: O(2^N)
- 공간: O(N)

## JS 구현 포인트

- 한 번 선택한 내용을 다시 바꿀 필요 없으므로 idx 사용
- 2가지 경우를 모두 호출하여 전체 탐색 가능하도록 함
