# [가장 먼 노드](https://school.programmers.co.kr/learn/courses/30/lessons/49189)

## 문제 요약

- 1번 노드에서 시작하여 다른 노드까지의 최단 거리 중
- 가장 멀리 떨어진 거리 값을 가지는 노드들의 개수를 구하는 문제

## 입력 구조

- n: 노드 개수, 2 이상 20000 이하
- edge: 간선 정보 배열
- 간선 개수: 1 이상 50000 이하
- [a, b]: 양방향 연결

## 문제 유형 분류

- BFS
- 그래프 탐색
- 최단거리

## 사고 과정

### 1. 왜 이 유형인가

- 가중치가 없는 그래프 + 최단거리 -> BFS

### 2. 왜 다른 접근은 안되는가

- DFS: 최단 거리 보장 안됨
- Dijkstra: 가중치가 없으므로 과한 접근
- Floyd-Warshall: O(n^3)로 비효율

## 문제 풀이

- [49189.js](./49189.js)

```
- 인접 리스트로 그래프 구성
- BFS로 1번 노드부터 거리 계산
- dist 배열에서 최대값 찾고 개수 세기
```

## 해결 전략

### 알고리즘 선택 이유

- 무가중치 그래프 최단거리 -> BFS
- 모든 노드까지 거리 계산 필요

### 자료구조 선택 이유

- 인접 리스트: 간선 수가 많을 때 효율적
- dist 배열: 방문 여부 + 거리 저장

## 시간/공간 복잡도

- 시간: O(n + m)
- 공간: O(n + m)

## JS 구현 포인트

- dist를 -1로 초기화하여 방문 여부와 거리 동시에 처리
- 인접 행렬이 아닌 인접 리스트 사용 (메모리 절약)
- shift 사용은 성능 이슈 가능
