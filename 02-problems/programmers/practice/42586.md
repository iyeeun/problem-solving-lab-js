# [기능개발](https://school.programmers.co.kr/learn/courses/30/lessons/42586)

## 문제 요약

- 각 작업은 완료까지 걸리는 시간이 있음
- 앞 작업이 완료되어야 뒤 작업도 배포 가능
- 배포 단위별로 몇 개씩 묶이는지 계산

## 입력 구조

- `progresses.length`: 100 이하
- 각 값: 1 이상 100 미만
- `speeds`: 1 이상 100 이하

## 문제 유형 분류

- 시뮬레이션

## 사고 과정

### 1. 왜 이 유형인가

- 작업 완료 시간을 기준으로 그룹을 만드는 문제

### 2. 왜 다른 접근은 안되는가

- 최적의 선택을 찾는 문제가 아니라 주어진 순서를 기준으로 묶는 문제임

## 문제 풀이

- [42586.js (21분 소요)](./42586.js)

```
각 작업의 완료일을 계산한 뒤
앞 작업 기준으로 묶어서 그룹 개수를 센다
```

## 해결 전략

### 알고리즘 선택 이유

- 각 작업은 독립적인 완료 시간 존재
- 하지만 배포는 앞 작업에 종속됨

### 자료구조 선택 이유

- 배열로 순서 유지

## 시간/공간 복잡도

- 시간: O(N)
- 공간: O(N)

## JS 구현 포인트

- `Math.ceil()`로 올림 처리
- 마지막 그룹까지 push
