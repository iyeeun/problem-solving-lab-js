# [가장 큰 수](https://school.programmers.co.kr/learn/courses/30/lessons/42746)

## 문제 요약

- 숫자 배열을 이어붙여 만들 수 있는 가장 큰 수를 구함
- 순서를 재배치 가능
- 결과는 문자열로 반환

## 입력 구조

- `numbers.length`: 1 이상 100000 이하
- 값 범위: 0 이상 1000 이하

## 문제 유형 분류

- 정렬

## 사고 과정

### 1. 왜 이 유형인가

- 정렬의 기준을 정의하는 문제

### 2. 왜 다른 접근은 안되는가

- 모든 경우를 만들면 O(N!): 불가능
- 숫자 크기 기준 정렬 안됨
- 이어 붙인 결과로 비교 필요

## 문제 풀이

1. [42746.js](./42746.js)

```
두 수를 이어붙인 결과를 기준으로 정렬
```

## 해결 전략

### 알고리즘 선택 이유

- 정렬을 통해 최적 순서를 구성

### 자료구조 선택 이유

- 배열 정렬 사용
- 문자열 비교 (이어붙이기)

## 시간/공간 복잡도

- 시간: O(N log N)
- 공간: O(N)

## JS 구현 포인트

- `"000"` -> `"0"` 처리 필요
