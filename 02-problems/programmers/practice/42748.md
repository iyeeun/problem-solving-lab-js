# [K번째수](https://school.programmers.co.kr/learn/courses/30/lessons/42748)

## 문제 요약

- commands 배열로 [i, j, k]가 배열로 주어짐. 각 command에 대한 아래 연산 결과를 배열로 반환
  - 배열의 특정 구간 `[i ~ j]`를 잘라
  - 정렬한 뒤
  - k번째 값을 구함

## 입력 구조

- array: 1 이상 100 이하
- array의 원소: 1 이상 100 이하
- commands의 길이: 1 이상 50 이하
- commands의 원소는 길이가 3 (i, j, k)

## 문제 유형 분류

- 단순 구현 (시뮬레이션)
- 정렬

## 사고 과정

### 1. 왜 이 유형인가

- 요구조건에 명시적으로 수행해야 하는 내용이 나와있음

### 2. 왜 다른 접근은 안되는가

- 입력이 크지 않기 때문에 단순 구현으로도 괜찮음

## 문제 풀이

1. [42748.js (4분 소요)](./42748.js)

```
for문을 돌면서 문제 요구사항 그대로 진행
answer에 결과 push
```

## 해결 전략

### 알고리즘 선택 이유

- 입력 크기가 작고 독립적으로 처리할 수 있음

### 자료구조 선택 이유

- 배열을 그대로 사용하되 slice를 이용해서 원본을 유지하고 계산된 값만 push하도록 함

## 시간/공간 복잡도

- 시간: O(commands \* n log n)
- 공간: O(n)

## JS 구현 포인트

- 인덱스 보정
- sort: 숫자 정렬 필수
- `commands.map()`으로 순회 대체 가능
