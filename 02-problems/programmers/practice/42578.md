# [의상](https://school.programmers.co.kr/learn/courses/30/lessons/42578)

## 문제 요약

- 각 의상 종류별로 최대 1개 선택
- 최소 1개는 반드시 착용
- 서로 다른 조합의 개수 구하기

## 입력 구조

- `clothes.length`: 1 이상 30 이하
- 각 원소: `[이름, 종류]`, 문자열 (길이 1 이상 20 이하, 알파벳 소문자 + \_)
- 이름 중복 없음

## 문제 유형 분류

- 조합
- 해시

## 사고 과정

### 1. 왜 이 유형인가

- 각 의상 종류에 대해 입을지 안 입을지 선택함
- 각 카테고리는 독립적인 선택을 가지며, 전체 경우의 수는 곱의 법칙으로 계산함

### 2. 왜 다른 접근은 안되는가

- 몇 가지의 조합이 나오는지 세는 것이고, 만드는 것이 아니므로  
  완전 탐색이나 DFS/백트래킹은 효율적이지 않음

## 문제 풀이

- [42578.js (27분 소요)](./42578.js)

```
각 카테고리별 개수를 세고
(개수 + 1)을 모두 곱한 뒤
아무것도 안 입는 경우(-1) 제거
```

## 해결 전략

### 알고리즘 선택 이유

- 각 카테고리에서 어떤 옷을 입을지 (안 입는 경우 포함) 하나 선택
- 해당 경우를 모두 곱하여 경우의 수를 계산하면 됨 (모두 안 입은 경우 하나 제외)

### 자료구조 선택 이유

- 간단한 카테고리 카운팅을 위한 객체 사용

## 시간/공간 복잡도

- 시간: O(N)
- 공간: O(K) (K = 종류 수)

## JS 구현 포인트

- `(num + 1)`: 선택 안함 포함
- `reduce`로 곱 연산 처리
- 마지막 `-1`으로 공집합 제거
