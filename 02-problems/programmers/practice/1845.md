# [폰켓몬](https://school.programmers.co.kr/learn/courses/30/lessons/1845)

## 문제 요약

- N마리의 폰켓몬 중 N/2개를 고를 때 가질 수 있는 폰켓몬 종류의 최대 개수

## 입력 구조

- nums.length == N : 1 이상 10000 이하의 자연수, 짝수
- 값 범위 : 1 이상 200000 이하의 자연수
- 중복 허용

## 문제 유형 분류

- Hash
- Greedy

## 사고 과정

1. 왜 이 유형인가

총 몇 종류의 폰켓몬이 존재하는지 확인  
-> 최대로 몇 종류를 가져갈 수 있는가?  
-> min(종류 개수, N/2)

2. 왜 다른 접근은 안되는가

- 조합을 통해 풀면 경우의 수가 너무 많기 때문에 일일히 세기 어려움
- 또한 문제 풀이에 중요한 것은 몇 종류가 있는지이기 때문에 단순히 종류만 카운트하면 됨

## 문제 풀이

1. [1845_1.js (6분 소요)](./1845_1.js)

```
- Map을 이용해서 종류를 카운팅함
- min(종류 개수, N/2) 반환
```

2. [1845.js](./1845.js)

```
- Set을 이용해서 더 간편하게 종류 개수 구함
```

## 해결 전략

### 알고리즘 선택 이유

- 선택 과정이 아니라 최대 가능한 값 계산
- upper bound는 두 가지: 종류 수, N/2  
  -> 둘 중 작은 값 선택

### 자료구조 선택 이유

#### Map

- 빈도 정보까지 필요할 때 사용
- 일반적인 해시 카운팅 패턴

#### Set

- 중복 제거만 필요할 때 사용

## 시간/공간 복잡도

- 시간: O(n)
- 공간: O(n)

## JS 구현 포인트

- Map
  - `(map.get(x) || 0) + 1` 패턴
  - map.size로 key 개수 확인
- Set
  - `new Set(nums)`로 한 번에 중복 제거
  - set.size 활용
- Math.min
  - 핵심 로직을 단순화
