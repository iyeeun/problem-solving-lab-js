# [최소직사각형](https://school.programmers.co.kr/learn/courses/30/lessons/86491)

## 문제 요약

- 명함의 가로 길이와 세로 길이 배열이 주어지면 모든 명함을 수납할 수 있는 가장 작은 지갑의 넓이를 반환

## 입력 구조

- 배열의 크기: 1 이상 10000 이하
- 배열의 원소: [w, h] 형식
- w와 h: 1 이상 1000 이하인 자연수

## 문제 유형 분류

- 구현
- Greedy

## 사고 과정

### 1. 왜 이 유형인가

- 각 명함의 가로/세로를 바꿀 수 있으므로 정렬을 통해 방향을 맞추면  
  가로 길이와 세로 길이의 최댓값을 구하면 되는 문제로 변함

### 2. 왜 다른 접근은 안되는가

- 모든 회전 경우 고려하는 시뮬레이션 풀이는  
  → 각 명함마다 2가지 → 2^n → 불가능

## 문제 풀이

- [86491.js (5분 소요)](./86491.js)

```
1. 가로와 세로 길이를 특정 방향으로 맞추기 위해 각 원소에 정렬 수행
2. 각 행의 최댓값을 찾고, 곱하여 너비 반환
```

## 해결 전략

### 알고리즘 선택 이유

- 각 명함은 독립적으로 회전 결정 가능

### 자료구조 선택 이유

- 2차원 배열 그대로 사용

## 시간/공간 복잡도

- 시간: O(n)
- 공간: O(1)

## JS 구현 포인트

- 각 행에 대한 정렬은 `forEach`로 수행
- 열을 뽑기 위해 map 사용 -> 순회하면서 직접 하는 게 성능적으로는 나을 수 있음
