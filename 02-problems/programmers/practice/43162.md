# [네트워크](https://school.programmers.co.kr/learn/courses/30/lessons/43162)

## 문제 요약

- 컴퓨터 간 연결 정보가 주어질 때 연결된 컴퓨터들의 그룹 (네트워크) 개수 구하기

## 입력 구조

- n: 컴퓨터 개수, 1 이상 200 이하
- computers: n x n 2차원 배열
- computers[i][j]가 1이면 i와 j는 연결됨
- computers[i][i]는 항상 1

## 문제 유형 분류

- DFS / BFS
- 그래프 탐색
- 연결 요소(Connected Components)

## 사고 과정

### 1. 왜 이 유형인가

- 각 컴퓨터는 노드이고 연결 여부가 edge이므로 그래프에서 연결 요소의 개수를 구하는 문제임

### 2. 왜 다른 접근은 안되는가

- Greedy: 전체 구조를 봐야 하므로 불가능
- DP: 상태 정의가 불명확
- 완전탐색: 결국 그래프 탐색으로 귀결됨

## 문제 풀이

- [43162.js](./43162.js)

```
- 방문하지 않은 노드에서 DFS 시작
- DFS 한 번 수행할 때마다 하나의 네트워크 완성
- DFS 호출 횟수 = 네트워크 개수
```

## 해결 전략

### 알고리즘 선택 이유

- 연결 요소 개수 문제 -> DFS/BFS
- 모든 노드를 방문하면서 그룹을 나눔

### 자료구조 선택 이유

- visited 배열: 방문 여부 관리
- 인접 행렬(computers): 연결 정보

## 시간/공간 복잡도

- 시간: O(n^2)
- 공간: O(n)

## JS 구현 포인트

- visited 체크를 DFS 초입에서 처리하여 중복 탐색 방지
