# [구명보트](https://school.programmers.co.kr/learn/courses/30/lessons/42885)

## 문제 요약

- 사람들의 몸무게 배열이 주어짐
- 보트는 최대 2명까지 탑승 가능
- 무게 제한 존재
- 모든 사람을 구출할 때 필요한 보트 최소 개수 구하기

## 입력 구조

- people: 길이 1 이상 50000 이하
- 각 원소: 40 이상 240 이하
- limit: 40 이상 240 이하

## 문제 유형 분류

- Greedy
- Two Pointer

## 사고 과정

### 1. 왜 이 유형인가

- 보트를 최대한 효율적으로 채우기 위해서는 가장 무거운 사람 + 가장 가벼운 사람
- 정렬 + 양쪽에서 접근으로 해결

### 2. 왜 다른 접근은 안되는가

- 완전탐색: 경우의 수 큼
- DFS/BFS: 조합 탐색 -> 시간 초과
- DP: 상태 정의 비효율

## 문제 풀이

- [42885.js (32분 소요)](./42885.js)

```
- 가장 무거운 사람을 기준으로 보트 사용
- 가장 가벼운 사람과 같이 탈 수 있으면 같이 태움
- 아니면 혼자 태움
```

## 해결 전략

### 알고리즘 선택 이유

- 무거운 사람은 반드시 먼저 처리해야 함
- 가장 가벼운 사람과 묶는 것이 최적

### 자료구조 선택 이유

- 정렬된 배열 + 투 포인터

## 시간/공간 복잡도

- 시간: O(N log N)
- 공간: O(1) (정렬 제외)

## JS 구현 포인트

- 정렬 필수
- `left <= right` 조건 중요
- 한 번에 right는 항상 감소
