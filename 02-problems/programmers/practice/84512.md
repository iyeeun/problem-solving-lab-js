# [모음사전](https://school.programmers.co.kr/learn/courses/30/lessons/84512)

## 문제 요약

- A, E, I, O, U 5개의 문자로 만들 수 있는 길이 1 ~ 5까지 모든 문자열을 사전순으로 나열
- 특정 word가 몇 번째인지 반환

## 입력 구조

- word 길이: 1 이상 5 이하
- A, E, I, O, U로만 이루어짐

## 문제 유형 분류

- 완전탐색 (DFS)
- 규칙 기반 Greedy도 가능

## 사고 과정

### 1. 왜 이 유형인가

- 가능한 모든 문자열이 `5^1 + 5^2 + 5^3 + 5^4 + 5^5 ≈ 3905`개라서 완전탐색 가능

### 2. 왜 다른 접근은 안되는가

- BFS: 가능은 하지만 순서 유지 + 구현 복잡도 증가
- DP: 상태 정의 애매
- 정렬: 어차피 다 만들어야 해서 의미 없음

## 문제 풀이

- [84512.js](./84512.js)

```
1. DFS로 길이 1~5 모든 문자열 생성
2. `res`에 사전순으로 저장됨
3. `indexOf`로 위치 찾기
```

## 해결 전략

### 알고리즘 선택 이유

- 경우의 수가 작아서 완전탐색 가능
- 사전순 = DFS 순서로 자연스럽게 생성됨

### 자료구조 선택 이유

- `path`: 현재 문자열 구성
- `res`: 전체 사전 저장

## 시간/공간 복잡도

- 시간: O(5^5) ≈ O(3125)
- 공간: O(5^5)

## JS 구현 포인트

- `indexOf` → O(N)
- 문자열 누적 대신 push/pop으로 백트래킹
