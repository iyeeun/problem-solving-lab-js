# [같은 숫자는 싫어](https://school.programmers.co.kr/learn/courses/30/lessons/12906)

## 문제 요약

- 배열 arr에서 연속적으로 나타나는 숫자는 제거한 배열 구하기

## 입력 구조

- 배열 arr의 크기: 1000000
- 배열 arr 원소의 크기 : 0부터 9까지의 정수

## 문제 유형 분류

- Stack

## 사고 과정

1. 왜 이 유형인가

- 연속인지 아닌지 판별  
  -> 이전 값과 현재 값이 같은지 판단  
  -> 앞의 값만 알고 해당 상태를 기반으로 진행  
  -> 스택/배열 기반 처리

2. 왜 다른 접근은 안되는가

- 직전 값 비교가 핵심이기 때문에 배열이 적합

## 문제 풀이

1. [12906.js (2분 소요)](./12906.js)

```
1. 결과 배열을 하나 만든다
2. 현재 값과 마지막 값 비교
3. 비어있거나 값이 다르면 push
```

## 해결 전략

### 알고리즘 선택 이유

- 모든 값을 알 필요가 없기 때문

### 자료구조 선택 이유

- push 및 마지막 값 접근만 하면 되기 때문에 배열을 스택처럼 사용

## 시간/공간 복잡도

- 시간: O(n)
- 공간: O(n)

## JS 구현 포인트

- `answer[answer.length - 1]`: 마지막 값 접근
- `answer.length === 0`: 초기 조건 처리
