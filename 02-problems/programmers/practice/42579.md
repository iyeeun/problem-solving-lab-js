# [베스트앨범](https://school.programmers.co.kr/learn/courses/30/lessons/42579)

## 문제 요약

- 장르별로 노래를 그룹화
- 많이 재생된 장르부터 선택
- 각 장르에서 재생 수 높은 노래 최대 2개 선택
- 재생 수가 같으면 인덱스가 작은 순

## 입력 구조

- genres: 길이 1 이상 10000 이하
- plays: 길이 동일
- 장르 종류: 100개 미만

## 문제 유형 분류

- 해시 (Map)
- 정렬
- 구현

## 사고 과정

### 1. 왜 이 유형인가

- 장르 별 총 재생 수 집계 + 정렬
- 그룹화 + 정렬 문제

### 2. 왜 다른 접근은 안되는가

- DFS/BFS: 탐색 문제 아님
- Greedy: 단순 선택이 아니라 정렬 기준 문제
- DP: 상태 누적 구조 아님

## 문제 풀이

- [42579.js (13분 소요)](./42579.js)

```
- 장르별 총 재생 수, 노래 집계
- 장르를 총 재생 수 기준으로 정렬
- 각 장르에서 재생 수 높은 순으로 노래 정렬 후 2개 선택
```

## 해결 전략

### 알고리즘 선택 이유

- 그룹화 후 정렬하는 문제
- Map을 활용한 집계 + 정렬

### 자료구조 선택 이유

- Map: 장르별 집계 및 그룹화
- 배열: 정렬 및 결과 구성

## 시간/공간 복잡도

- 시간: O(N log N)
- 공간: O(N)

## JS 구현 포인트

- `entries()`로 정렬
- `slice(0, 2)`로 상위 2개 선택
