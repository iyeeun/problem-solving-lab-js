# [프로세스](https://school.programmers.co.kr/learn/courses/30/lessons/42587)

## 문제 요약

- 각 프로세스의 우선순위가 주어질 때 원하는 프로세스가 몇 번째로 실행되는지 찾기
- 프로세스 실행 규칙

```
  1. 실행 대기 큐(Queue)에서 대기중인 프로세스 하나를 꺼냅니다.
  2. 큐에 대기중인 프로세스 중 우선순위가 더 높은 프로세스가 있다면 방금 꺼낸 프로세스를 다시 큐에 넣습니다.
  3. 만약 그런 프로세스가 없다면 방금 꺼낸 프로세스를 실행합니다.
  4. 한 번 실행한 프로세스는 다시 큐에 넣지 않고 그대로 종료됩니다.
```

## 입력 구조

- `priorities.length`: 1 이상 100 이하
- 우선순위 값: 1 이상 9 이하의 정수 (클수록 높은 우선순위)
- `location`: 0 이상 (대기 큐에 있는 프로세스 수 - 1) 이하

## 문제 유형 분류

- 큐
- 시뮬레이션

## 사고 과정

### 1. 왜 이 유형인가

- 큐의 동작을 시뮬레이션 하는 문제
- 앞에서 꺼내고 조건에 따라 뒤로 보냄

### 2. 왜 다른 접근은 안되는가

- 순서 유지 중요 -> 정렬 X
- 위치 변화 추적 필요 -> 단순 최댓값 추적 X

## 해결 전략

### 알고리즘 선택 이유

- 문제 자체가 큐의 동작을 시뮬레이션하는 것임

### 자료구조 선택 이유

- 배열: max 기준으로 자르기 (회전 구현)
- 동작을 그대로 수행 (큐 + shift/push)

## 시간/공간 복잡도

- 시간: O(N^2)
- 공간: O(N)

## JS 구현 포인트

- 더 큰 값이 있는지 확인 -> `some()`
- `priorities.map((p, i) => ({ p, i }));`으로 간단하게 인덱스 페어 생성
- `shift()`는 맨 앞의 원소를 삭제해서 반환
