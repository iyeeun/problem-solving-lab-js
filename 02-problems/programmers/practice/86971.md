# [전력망을 둘로 나누기](https://school.programmers.co.kr/learn/courses/30/lessons/86971)

## 문제 요약

- 트리 구조의 전력망에서 전선 하나를 끊는다
- 두 개의 네트워크로 나뉨
- 각 네트워크의 노드 개수 차이를 최소화
- 그 차이의 최솟값을 구하는 문제

## 입력 구조

- n: 2 이상 100 이하
- wires: 길이 n-1인 배열 (트리 구조 보장)
- 각 원소: [v1, v2]

## 문제 유형 분류

- DFS / 그래프 탐색
- 완전탐색
- 트리

## 사고 과정

### 1. 왜 이 유형인가

- 트리에서 edge 하나를 끊으면 두 개의 연결 요소가 생기므로
- edge 하나 제거 -> 크기 계산을 반복

### 2. 왜 다른 접근은 안되는가

- Greedy: 어떤 간선을 끊을지 기준 없음
- DP: 트리 분할 상태 정의가 복잡
- BFS: 가능하지만 DFS와 본질 동일

## 문제 풀이

- [86971.js (25분 소요)](./86971.js)

```
- 각 간선을 하나씩 제거한다고 가정
- 제거된 상태에서 하나의 노드에서 DFS 수행
- 두 그룹의 크기 차이를 계산
```

## 해결 전략

### 알고리즘 선택 이유

- 모든 간선을 하나씩 제거해야 하므로 완전탐색
- 각 경우마다 연결 요소 크기를 구해야 하므로 DFS

### 자료구조 선택 이유

- visited 배열: 방문 여부 관리
- 인접 리스트: 탐색 효율 향상

## 시간/공간 복잡도

- 시간: O(n^2)
- 공간: O(n)

## JS 구현 포인트

- 인접 리스트로 탐색 최적화
- DFS에서 count를 return하도록 설계
