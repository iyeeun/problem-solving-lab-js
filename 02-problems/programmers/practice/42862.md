# [체육복](https://school.programmers.co.kr/learn/courses/30/lessons/42862)

## 문제 요약

- 일부 학생들이 체육복을 도난당함
- 일부 학생은 여벌 보유
- 앞/뒤 번호 학생에게만 빌려줄 수 있음
- 최대한 많은 학생이 수업 참여 가능하도록 해야 함

## 입력 구조

- n: 2 이상 30 이하
- lost: 1명 이상 n명 이하, 중복 없음
- reserve: 1명 이상 n명 이하, 중복 없음
- lost이면서 reserve는 가능

## 문제 유형 분류

- Greedy
- 시뮬레이션

## 사고 과정

### 1. 왜 이 유형인가

- 누가한테 빌려줄지 전부 탐색하는 것이 아니라,
- 가능하면 빌려주도록 함

### 2. 왜 다른 접근은 안되는가

- 빌릴 수 있는 조건 자체가 앞이나 뒤 뿐이고,
- 앞에서 빌려준다고 해서 뒤에 영향을 주는 부분이 거의 없음
- 미래를 고려해서 미루는 전략이 의미 없음

## 문제 풀이

- [42862.js (8분 소요)](./42862.js)

```
1. 각 학생들의 체육복 개수를 의미하는 배열을 만듦
  - 이 때 0과 n + 1에도 0을 추가하여 패딩을 추가함
2. lost 배열을 돌면서 도난을 표시하고 (1 감소), reserve 배열을 돌면서 여분을 표시함 (1 증가)
3. count 배열을 돌면서 체육복 개수가 0인 경우에 앞이나 뒤가 여분이 있으면 가져옴
4. 체육복이 있는 학생의 수를 반환함
```

## 해결 전략

### 알고리즘 선택 이유

- 학생 간 영향 범위가 제한됨
- 각 학생은 한 번만 판단하면 됨

### 자료구조 선택 이유

- 배열을 통해 상태를 직관적으로 관리할 수 있음
  - 인덱스 == 학생 번호
  - 값 == 체육복 개수

## 시간/공간 복잡도

- 시간: O(n)
- 공간: O(n)

## JS 구현 포인트

- `new Array(n + 2).fill(1)`: 경계 처리 쉽게 하기 위한 패딩
