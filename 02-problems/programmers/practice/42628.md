# [이중우선순위큐](https://school.programmers.co.kr/learn/courses/30/lessons/42628)

## 문제 요약

- 숫자를 삽입/삭제하는 큐
- 최댓값 삭제 / 최솟값 삭제 둘 다 지원
- 모든 연산 후 [최댓값, 최솟값] 반환

## 입력 구조

- operations: 길이 1 이상 1000000 이하
  - "I 숫자", "D 1", "D -1"

## 문제 유형 분류

- Heap
- Lazy Deletion
- 자료구조 구현

## 사고 과정

### 1. 왜 이 유형인가

- 최솟값, 최댓값에 빠르게 접근해서 연산을 수행해야 함
- => minHeap + maxHeap

### 2. 왜 다른 접근은 안되는가

- 정렬 배열: 삽입/삭제 O(N) -> 시간 초과
- 단일 heap: 한쪽 값만 빠름
- Set: 순서 유지 안됨

## 문제 풀이

- [42628.js (40분 소요)](./42628.js)

```
- Heap 정의
- minHeap, maxHeap을 모두 가지면서 살아있는 값을 확인하는 count Map 정의
- 힙에서 값을 꺼낼 때 죽은 값 정리 (lazy deletion)
```

## 해결 전략

### 알고리즘 선택 이유

- 양쪽 끝값 접근 -> heap 2개 필요
- 삭제 동기화 -> lazy deletion 필요

### 자료구조 선택 이유

- Heap: 최댓값/최솟값 접근
- Map: 실제 데이터 개수 관리

## 시간/공간 복잡도

- 시간: O(N log N)
- 공간: O(N)

## JS 구현 포인트

- clean 함수로 유효하지 않은 값 제거
- Map으로 중복 개수 관리
- pop 전에 항상 clean 호출
