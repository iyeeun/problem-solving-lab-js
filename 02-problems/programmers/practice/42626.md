# [더 맵게](https://school.programmers.co.kr/learn/courses/30/lessons/42626)

## 문제 요약

- 가장 작은 두 값을 꺼내 섞음
- 모든 음식의 스코빌 지수를 K 이상으로 만드는 최소 횟수 반환

## 입력 구조

- `scoville.length`: 2 이상 1000000 이하
- 값 범위: 0 이상 1000000 이하
- K: 0 이상 1000000000 이하

## 문제 유형 분류

- Heap

## 사고 과정

### 1. 왜 이 유형인가

- 매번 가장 작은 값 2개를 꺼내는 문제

### 2. 왜 다른 접근은 안되는가

- 정렬 -> 매 반복마다 O(N log N) -> O(N^2 log N) -> 안됨

1. [42626_1.js (약 22분 소요)](./42626_1.js)

```
sort와 shift로 최솟값 꺼내기 -> 시간 초과
```

```
1. MinHeap 정의
2. MinHeap 이용해서 가장 작은 값 2개 꺼냄 + 새로운 값 추가 반복
```

## 해결 전략

### 알고리즘 선택 이유

- 매번 가장 작은 값 2개 필요
- 정렬보다 빠른 자료구조 필요 -> Heap

### 자료구조 선택 이유

- MinHeap
  - 최소값 추출 O(log N)
  - 삽입 O(log N)

## 시간/공간 복잡도

- 시간: O(N log N)
- 공간: O(N)

## JS 구현 포인트

- JS는 heap 내장 없음 -> 직접 구현 필요
- index 기반 트리 구조
