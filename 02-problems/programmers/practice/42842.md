# [카펫](https://school.programmers.co.kr/learn/courses/30/lessons/42842)

## 문제 요약

- 전체 카펫은 갈색 테두리 1줄 + 내부 노란색으로 구성
- 갈색 개수와 노란색 개수가 주어질 때 카펫의 가로, 세로 길이를 구하는 문제

## 입력 구조

- brown: 8 이상 5000 이하
- yellow: 1 이상 2000000 이하
- 가로 길이 >= 세로 길이

## 문제 유형 분류

- 완전탐색 (약수 탐색)
- 수학 (방정식)

## 사고 과정

### 1. 왜 이 유형인가

- w \* h = brown + yellow
- 약수를 이용해서 후보 탐색 -> 완전탐색(약수)

### 2. 왜 다른 접근은 안되는가

- BFS/DFS: 구조 탐색 문제가 아님
- Greedy: 선택 기준 없음
- DP: 상태 누적 구조 아님

## 문제 풀이

- [42842.js (11분 소요)](./42842.js)

```
yellwo = (w - 2) * (h - 2)이므로
1. yellow의 약수를 찾고
2. 해당 w, h가 전체 개수와 같은지 확인하여 반환
```

## 해결 전략

### 알고리즘 선택 이유

- yellow 기준으로 내부 영역부터 계산
- 약수 탐색 범위를 sqrt(yellow)까지 줄임

### 자료구조 선택 이유

- 변수만으로 계산 (추가 자료구조 없음)

## 시간/공간 복잡도

- 시간: O(sqrt(yellow))
- 공간: O(1)

## JS 구현 포인트

- 약수 탐색은 sqrt(n)까지만 진행해도 됨
