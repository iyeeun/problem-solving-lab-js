# [주식가격](https://school.programmers.co.kr/learn/courses/30/lessons/42584)

## 문제 요약

- 각 시점에서 가격이 떨어지지 않은 기간(초) 계산
- 가격이 처음으로 떨어지는 시점까지 카운트

## 입력 구조

- `prices.length`: 2 이상 100000 이하
- 가격: 1 이상 10000 이하인 자연수

## 문제 유형 분류

- 시뮬레이션
- 스택

## 사고 과정

### 1. 왜 이 유형인가

- 조건대로 구현하여 찾기
- 언제 자신보다 작은 값이 나오는지 찾기

### 2. 왜 다른 접근은 안되는가

- 이전 값을 기억하면서 비교하는 것이기 때문에 스택이 적절함

## 문제 풀이

1. [42584_1.js (12분 소요)](./42584_1.js)

```
각 시점마다 뒤를 끝까지 탐색
```

2. [42584.js (12분 소요)](./42584.js)

```
1. stack에 아직 안 떨어진 수 쌓음
2. 현재 값이 더 작으면 이전 pop하면서 결과 계산
3. 끝까지 안 떨어진 수는 따로 처리
```

## 해결 전략

- 이전 값 기억 -> 스택

## 시간/공간 복잡도

- 시간: O(N^2) / O(N)
- 공간: O(N)

## JS 구현 포인트

- 스택에 인덱스를 저장
- 마지막 남은 것 처리
