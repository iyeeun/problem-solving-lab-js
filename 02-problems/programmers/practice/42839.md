# [소수 찾기](https://school.programmers.co.kr/learn/courses/30/lessons/42839)

## 문제 요약

- 주어진 숫자 조각들을 조합해서 만들 수 있는 모든 수 중 소수의 개수를 구하는 문제
- 같은 숫자라도 다른 순서면 다른 수로 취급
- 중복 숫자 조합은 제거해야 함

## 입력 구조

- numbers: 숫자로 이루어진 문자열
- 길이: 1 이상 7 이하
- 각 문자: 0 이상 9 이하의 숫자

## 문제 유형 분류

- DFS
- 완전탐색
- 백트래킹
- 에라토스테네스의 체

## 사고 과정

### 1. 왜 이 유형인가

- 가능한 모든 숫자 조합을 만들어야 하므로 완전탐색 필요
- 순열 형태로 모든 경우를 탐색 -> DFS + 백트래킹
- 생성된 숫자가 소수인지 빠르게 판단해야 하므로 미리 소수 배열 생성

### 2. 왜 다른 접근은 안되는가

- Greedy: 최적 선택 기준이 없음 (모든 경우 확인 필요)
- DP: 이전 결과를 재활용할 수 있는 구조가 아님
- BFS: 탐색 방식 차이일 뿐, DFS가 더 직관적
- Two Pointer: 정렬 기반 구간 탐색 문제 아님

## 문제 풀이

- [42839.js](./42839.js)

```
1. 에라토스테네스의 체로 최대까지 소수 여부 미리 계산
2. numbers를 배열로 변환
3. DFS로 모든 순열 생성 (길이 1 이상)
4. 생성된 숫자들을 배열에 저장
5. Set으로 중복 제거
6. 각 숫자가 소수인지 확인 후 개수 카운트
```

## 해결 전략

### 알고리즘 선택 이유

- 숫자 길이가 최대 7
  - 최대 경우의 수 7! 정도로 모든 순열을 생성해도 충분히 계산 가능
- 소수 판별을 매번 하면 느리므로 미리 계산

### 자료구조 선택 이유

- Set으로 동일 숫자 제거

## 시간/공간 복잡도

- 시간: O(N! + M log log M)
  - N!: 순열 생성
  - M: 최대 숫자 범위 (10000000)에서 체 생성
- 공간: O(M + N!)

## JS 구현 포인트

- DFS에서 `cur.length > 0`일 때만 결과 추가
- Set으로 중복 제거
- 에라토스테네스의 체 범위는 만들 수 있는 최대 숫자보다 크게 설정
