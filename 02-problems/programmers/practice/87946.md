# [피로도](https://school.programmers.co.kr/learn/courses/30/lessons/87946)

## 문제 요약

- 현재 피로도 k에서 시작
- 각 던전은 최소 필요 피로도와 소모 피로도를 가짐
- 조건을 만족하는 던전만 선택 가능
- 최대 몇 개의 던전을 탐험할 수 있는지 구하는 문제

## 입력 구조

- k: 1 이상 5000 이하
- dungeons: 길이 1 이상 8 이하
- 각 원소: [최소 필요 피로도, 소모 피로도]

## 문제 유형 분류

- DFS / 완전탐색
- 순열 (Permutation)

## 사고 과정

### 1. 왜 이 유형인가

- 던전 개수가 최대 8개이므로 모든 순서(순열) 시도 가능

### 2. 왜 다른 접근은 안되는가

- Greedy: 최적 선택 기준 없음 (반례 존재)
- DP: 상태가 순서 + 피로도로 복잡
- BFS: 가능하지만 상태 관리가 더 복잡

## 문제 풀이

- [87946.js (49분 소요)](./87946.js)

```
- 현재 피로도에서 갈 수 있는 던전을 모두 시도
- 방문 시 visited 처리 후 재귀
- 더 이상 못 가는 시점까지 탐색
- 매 단계마다 최대 개수 갱신
```

## 해결 전략

### 알고리즘 선택 이유

- 던전 방문 순서에 따라 결과가 달라짐
- 모든 순서를 탐색해야 하므로 DFS

### 자료구조 선택 이유

- visited 배열: 중복 방문 방지
- 별도 path 없이 count로 상태 압축

## 시간/공간 복잡도

- 시간: O(N!)
- 공간: O(N)

## JS 구현 포인트

- visited를 이용한 백트래킹
- 매 호출마다 answer 갱신
