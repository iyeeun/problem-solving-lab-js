# [게임 맵 최단거리](https://school.programmers.co.kr/learn/courses/30/lessons/1844)

## 문제 요약

- 2차원 맵에서 (0, 0)에서 (n-1, m-1)까지 이동
- 이동은 상하좌우 한 칸씩 가능
- 0은 벽, 1은 이동 가능
- 최단 거리로 도착할 때 지나가는 칸의 개수 반환
- 도달할 수 없으면 -1 반환

## 입력 구조

- maps: n x m 2차원 배열
- n, m: 각각 1 이상 100 이하
- 값: 0 또는 1
- 시작 위치: (0, 0)
- 도착 위치: (n-1, m-1)

## 문제 유형 분류

- BFS
- 최단거리 (그래프 탐색)
- 격자 탐색

## 사고 과정

### 1. 왜 이 유형인가

- 최단 거리를 구하는 문제이면서 모든 비용이 동일하기 때문에 BFS

### 2. 왜 다른 접근은 안되는가

- DFS: 최단 거리 보장 안됨, 모든 경로 탐색 필요
- Greedy: 지역 선택이 전체 최적을 보장하지 않음
- DP: 경로가 여러 방향으로 분기되어 상태 정의 어려움

## 문제 풀이

- [1844.js](./1844.js)

```
- 방문하지 않은 길(1)을 -1로 바꿔 방문 여부 구분
- BFS로 탐색하면서 거리 누적
- 도착 지점 값이 최단 거리
```

## 해결 전략

### 알고리즘 선택 이유

- 모든 간선의 비용이 동일 + 최단 거리 문제 -> BFS

### 자료구조 선택 이유

- 큐: BFS 진행
- maps 배열 자체를 거리 + 방문 체크로 활용

## 시간/공간 복잡도

- 시간: O(n \* m)
- 공간: O(n \* m)

## JS 구현 포인트

- maps를 직접 수정하여 visited 배열을 따로 만들지 않음
- dx, dy로 방향 처리
