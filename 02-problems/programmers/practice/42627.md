# [디스크 컨트롤러](https://school.programmers.co.kr/learn/courses/30/lessons/42627)

## 문제 요약

- 작업 요청 시점과 소요 시간이 주어짐
- 한 번에 하나의 작업만 수행 가능
- 대기 큐에서 우선순위가 높은 작업 선택
  - 소요시간 짧은 순
  - 요청 시각 빠른 순
  - 번호 작은 순
- 모든 작업의 최소 반환 시간 평균 구하기

## 입력 구조

- jobs: 길이 1 이상 500 이하
  - 각 원소: [요청 시점, 소요 시간]
- 요청 시점: 0 이상 1000 이하
- 소요 시간: 1 이상 1000 이하

## 문제 유형 분류

- Heap (Priority Queue)
- Greedy
- 시뮬레이션

## 사고 과정

### 1. 왜 이 유형인가

- 현재 시점에서 실행 가능한 작업 중 가장 짧은 작업을 선택
- => SJF (Shortest Job First)
- => Priority Queue

### 2. 왜 다른 접근은 안되는가

- 완전탐색: 모든 순서 탐색 불가능
- DFS/BFS: 상태 공간 너무 큼
- 단순 정렬: 동적으로 작업이 추가됨

## 문제 풀이

- [42627.js (60+분 소요)](./42627.js)

```
- 시간 기준으로 jobs 정렬
- 현재 시간까지 들어온 작업들을 heap에 넣음
- heap에서 가장 짧은 작업 선택
- 작업 완료 시간 기준으로 반환 시간 계산
```

## 해결 전략

### 알고리즘 선택 이유

- 항상 가장 짧은 작업을 선택해야 평균 최소화
- 동적으로 작업이 추가되므로 heap 필요

### 자료구조 선택 이유

- MinHeap: 소요 시간 기준 우선순위 관리

## 시간/공간 복잡도

- 시간: O(N log N)
- 공간: O(N)

## JS 구현 포인트

- jobs를 요청 시간 기준으로 정렬
- **heap이 비었을 때 시간 점프 처리**
- 복잡한 조건 처리를 위한 compare Heap
