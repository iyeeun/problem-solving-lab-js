# [올바른 괄호](https://school.programmers.co.kr/learn/courses/30/lessons/12909)

## 문제 요약

- `'('` 와 `')'`로 이루어진 문자열
- 올바른 괄호 문자열인지 판단
  - 열리면 반드시 닫혀야 함
  - 순서도 맞아야 함

## 입력 구조

- `s.length`: 100000 이하의 자연수
- 문자: `'('`, `')'`만 존재

## 문제 유형 분류

- 스택
- 문자열

## 사고 과정

### 1. 왜 이 유형인가

- 괄호는 최근에 열린 게 먼저 닫혀야 함 -> LIFO 구조 -> 스택

### 2. 왜 다른 접근은 안되는가

- 순서와 개수를 모두 파악해야 함 (페어 매칭)

## 문제 풀이

- [12909.js (4분 소요)](./12909.js)

```
'('이면 push
')'이면 pop (없으면 false)
마지막에 stack 비어있으면 true
```

## 해결 전략

### 알고리즘 선택 이유

- 괄호 매칭 문제이기 때문에 LIFO -> 스택으로 해결함
- 단순 매칭 확인이기 때문에 카운터로도 해결 가능함

### 자료구조 선택 이유

- 문제와 구조적으로 가장 직관적으로 매칭됨
- 카운터 사용 시 메모리 최적화 가능

## 시간/공간 복잡도

- 시간: O(N)
- 공간: O(N)

## JS 구현 포인트

- early return으로 불필요 연산 제거
